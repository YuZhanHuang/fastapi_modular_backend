# 依賴注入流程詳解

本文檔說明從 HTTP 請求到 Service 實例創建的完整依賴注入流程（dependency-injector 架構）。

---

## 完整流程圖

```
HTTP 請求
    ↓
[步驟 1] FastAPI 路由處理（carts.py）
    ↓
[步驟 2] FastAPI Depends() 觸發 CartServiceDep
    ↓
[步驟 3] inject_service(container.services.cart_service)
    ↓
[步驟 4] Depends(get_session) 建立 request-scoped Session
    ↓
[步驟 5] container.services.cart_service(session=session)
    ↓
[步驟 6] ServiceContainer 建立 CartRepositoryImpl(session)
    ↓
[步驟 7] 組裝 CartService(cart_repo=...) 並返回
    ↓
[步驟 8] 路由處理函數執行業務邏輯
    ↓
[步驟 9] get_session finally 關閉 Session
```

---

## 詳細步驟說明

### 階段一：HTTP 請求進入

#### 步驟 1：FastAPI 路由處理

**位置：** `api/carts.py`

```python
CartServiceDep = Annotated[
    CartService,
    Depends(inject_service(get_container().services.cart_service)),
]

@router.get("/cart")
def get_cart(user_id: str, service: CartServiceDep) -> ...:
    cart = service.get_cart(user_id)
```

FastAPI 發現 `service` 參數綁定 `CartServiceDep`，觸發依賴注入。

---

### 階段二：FastAPI 橋接層

#### 步驟 2–3：inject_service 執行

**位置：** `api/deps.py`

```python
def inject_service(provider: providers.Factory[T]) -> Callable[..., T]:
    def _dependency(session: Session = Depends(get_session)) -> T:
        return provider(session=session)
    return _dependency
```

- `get_session` 是唯一 HTTP 專用的 `Depends`，負責 yield/close Session
- `inject_service` 將 session 傳入 dependency-injector Factory

#### 步驟 4：get_session 建立 Session

**位置：** `infra/db/session.py`

```python
def get_session() -> Generator[Session, None, None]:
    db = get_container().infra.session_factory()()
    try:
        yield db
    finally:
        db.close()
```

`session_factory` 來自 `InfrastructureContainer` Singleton。

---

### 階段三：IoC Container 組裝

#### 步驟 5–7：Service Factory 解析

**位置：** `infra/containers/services.py`

```python
def _create_cart_service(cart_repository_factory, session: Session) -> CartService:
    return CartService(cart_repo=cart_repository_factory(session=session))

cart_service = providers.Factory(
    _create_cart_service,
    cart_repository_factory=repos.cart_repository.provider,
    session=providers.Dependency(instance_of=Session),
)
```

**位置：** `infra/containers/repositories.py`

```python
cart_repository = providers.Factory(
    CartRepositoryImpl,
    session=providers.Dependency(instance_of=Session),
)
```

呼叫鏈：

1. `cart_service(session=session)` 觸發 `_create_cart_service`
2. `_create_cart_service` 呼叫 `cart_repository_factory(session=session)`
3. 建立 `CartRepositoryImpl(session)` 並注入 `CartService`

---

### 階段四：應用啟動

**位置：** `application/app.py`

```python
def create_app(...) -> FastAPI:
    app_container = init_container(container)
    app = AppFactory.create(api_type)
    app.state.container = app_container
    return app
```

**位置：** `infra/containers/__init__.py`

```python
def init_container(container=None) -> ApplicationContainer:
    _container = ApplicationContainer()
    _container.config.from_dict({
        "database_url": settings.DATABASE_URL,
        "redis_url": f"redis://{settings.REDIS_HOST}:{settings.REDIS_PORT}/0",
    })
    _container.init_resources()
    return _container
```

---

## Scope 策略

| 元件 | Provider 類型 | 生命週期 |
|---|---|---|
| `db_engine`, `session_factory`, `redis` | Singleton | 進程級 |
| `Session` | 外部注入 / Resource | 請求級 |
| `*RepositoryImpl` | Factory | 每次解析新建 |
| `*Service` | Factory | 每次解析新建 |

---

## 新增實體時

以 `User` 為例，需在 Container 明確註冊（取代舊的自動發現）：

1. `repositories.py` 加 `user_repository = providers.Factory(UserRepositoryImpl, session=...)`
2. `services.py` 加 `user_service = providers.Factory(...)`
3. `api/users.py` 加 `UserServiceDep = Annotated[..., Depends(inject_service(get_container().services.user_service))]`

詳見 [IoC Container 模組](./Wiring%20模組-自動化依賴注入.md)。
